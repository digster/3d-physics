# 2026-07-12 ‚ÄĒ Project kickoff & Milestone 1 (Part I)

## What this project is
A tutorial-based course for learning **3D physics** by building a small engine
from scratch in **SDL3 + C++20**. Each chapter = a runnable demo
(`chapters/NN-*/main.cpp`) + an in-depth static HTML doc page (`docs/`, no build
step). No physics or graphics engine dependency ‚ÄĒ we write the integrators,
collision, solver, and even a software 3D renderer ourselves.

Full plan: 20 chapters across 7 parts (Foundations ‚Üí Particles ‚Üí Rigid Bodies ‚Üí
Collision Detection ‚Üí Collision Response ‚Üí Joints ‚Üí Soft Bodies), delivered
**checkpoint per part**. Rendering is a **software pipeline** on
`SDL_RenderGeometry` (chosen so the 3D math is tutorial content, not a black box).

## Gotchas / learnings (see also LEARNINGS.md)
- Clang rejects `Config cfg = {}` default arg when the type has a `std::string`
  NSDMI *inside the enclosing class*; use a delegating default ctor instead.
- doctest's CMake needs `CMAKE_POLICY_VERSION_MINIMUM=3.5` under CMake 4.x.
- The in-app automated browser reports pages as `document.hidden`, which pauses
  `requestAnimationFrame` ‚ÄĒ widgets are fine for real users; to verify in the
  pane, patch rAF to `setTimeout`. Also the pane's screenshot compositor can go
  blank right after a *programmatic* scroll (scroll to top and it repaints).

### CRITICAL LEARNING ‚ÄĒ rotational energy leak (see LEARNINGS.md)
First rigid-body integrator used the naive quaternion step
`q += ¬Ĺ ŌČ‚äóq ¬∑ dt; normalize`. It conserved L exactly but **injected energy**: a
free spin sped up unboundedly (KE 11‚Üí21 over 10 s) instead of flipping ‚ÄĒ the
Chapter 5 explicit-Euler leak, in rotational form. Fixed with an **exponential-map
midpoint** step in `RigidBody::integrate` (sample ŌČ at the step midpoint, rotate
by the exact angle |ŌČ|¬∑dt). Then KE holds steady and the Dzhanibekov flip appears
correctly. Verified by a standalone probe and the unit test. The flip's tell is
the *body-fixed spin axis reversing in world space* ‚ÄĒ NOT the world-frame ŌČ_y
(which stays ~constant for the intermediate spin); don't test the wrong signal.

### CRITICAL LEARNING ‚ÄĒ GJK/EPA robustness (see LEARNINGS.md)
EPA is winding-sensitive. It diverged (hundreds of faces, no convergence) because
(a) the initial tetra face set wasn't consistently wound, and (b) `addFace` was
`std::swap`-ing indices to orient normals ‚ÄĒ which breaks the horizon-edge
cancellation. Fix: consistently-wound face set `{0,1,2},{0,3,1},{0,2,3},{1,3,2}`
and flip only the normal *vector*. Also: GJK's box simplex can be degenerate ‚Üí
"blow up" to a real tetra before EPA. And EPA's outward normal is A‚ÜíB, so negate
for the B‚ÜíA Contact convention. All cross-checked against SAT.

### CRITICAL LEARNING ‚ÄĒ Baumgarte energy leak (see LEARNINGS.md)
Summing the restitution and Baumgarte biases made a dropped ball bounce HIGHER
than it fell (effective e ~0.96 for a set 0.8) ‚ÄĒ deep single-step penetration
inflates the baumgarte term on top of the real bounce. Fix: `max(restitution,
baumgarte)`. Also: clamp the *accumulated* normal impulse (not the delta);
spawn bodies one-at-a-time to avoid overlap-launch; test stacks with sleeping ON
(basic sequential impulse drifts on tall stacks otherwise).
