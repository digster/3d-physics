Appendix C

Engine Anatomy

Twenty chapters build up a lot of code. This is the map: how the pieces fit, what runs in a single step, and where to look for each idea. If you're returning to the codebase after finishing the course, start here.

Three layers

Everything sorts into three layers, from most reusable to most specific. Each common/ and physics/ file has a header comment naming the chapter that introduced it, so the code can be read in tutorial order.

The split exists so the course grows by addition, not rewriting: chapter 20 reuses the exact camera and renderer from chapter 4, and physics/ depends only on the header-only math — no SDL — so it's unit-tested in isolation.

Two decisions that shape everything

A software renderer We transform geometry on the CPU and hand SDL flat 2D triangles, rather than using a GPU shader pipeline. That's deliberate: the model → world → view → clip → screen journey is chapters 3–4, so it belongs in code you own. The cost — the painter's algorithm can mis-sort interpenetrating triangles — is exactly the state the physics is about to fix, so it's a fair trade for a tutorial.
Store angular momentum, not angular velocity RigidBody keeps L and derives ω=I−1L each step, integrating orientation with an exponential-map midpoint. This conserves energy for free rotation (so the Dzhanibekov flip appears instead of a spin-up to infinity) — a direct echo of the Chapter 5 integration lesson.

How to read the code

In tutorial order. Every file's header comment names the chapter that introduces it, so you can follow the same arc the docs do: start in common/math/vec3.hpp, walk through the renderer, then read physics/ in the numeric order of the files' "introduced in" chapters, ending at softbody.cpp. The chapter demos in chapters/ show each piece in use, small enough to read in one sitting.

Build & verify cmake -B build && cmake --build build -j builds every demo into build/bin/; ctest --test-dir build runs the unit tests; and every demo takes --selftest (render a few frames headless, save a BMP, exit 0) for a quick smoke test. See ARCHITECTURE.md and LEARNINGS.md in the repo for the same map plus the bugs we hit along the way.
