SDL3 · C++ · No engine, no magic

Learn 3D physics by building it from scratch.

A chapter-by-chapter course that takes you from “what is a vector” to a small, real rigid-body engine with collisions, friction, joints, and soft bodies. Every idea comes as a runnable demo and a page like this that explains the intuition, the math, and the code.

Who this is for

You should be comfortable reading C++. Beyond that, no prior physics, graphics, or advanced math is assumed — we reintroduce every idea (vectors, matrices, a bit of calculus intuition) as we need it, and gloss the jargon the first time it appears. Read the chapters in order; each one builds directly on the last. If a page ever feels like it skipped a step, that’s a bug in the writing — the goal is that a motivated beginner can follow every line.

Getting the code running

You need a C++20 compiler and CMake. SDL3 is optional — if it isn’t installed, the build fetches and compiles it for you.

# macOS: brew install sdl3 cmake      (Linux: use your package manager)
cmake -B build
cmake --build build -j

# run any chapter (binaries land in build/bin):
./build/bin/ch01_hello_sdl3
./build/bin/ch05_integration

# run the unit tests:
ctest --test-dir build --output-on-failure
Standard controls Every 3D demo shares them: drag orbit · wheel zoom · Space pause · . single-step · R reset · 1–9 scene variant · W wireframe · H HUD · Esc quit.

How the project fits together

Three layers: each chapter’s tiny main.cpp sets up a scene and leans on a shared framework (common/ — the math, the renderer, the loop) and, from Part II on, a from-scratch engine (physics/). The reasoning behind the structure is in ARCHITECTURE.md.

The philosophy No Bullet, no Jolt, no PhysX — and no graphics engine either. We write the integrators, the collision detection, the solver, and even the 3D renderer ourselves, because the point is to understand why each one works.
