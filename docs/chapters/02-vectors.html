Vectors

A vector is an arrow: it has a length and a direction. That’s the whole idea. The power comes from two operations — the dot product and the cross product — that answer “how aligned are these?” and “what’s perpendicular to both?”. Almost everything in 3D physics is built from those two questions.

Goal

Build a reliable mental picture of vectors and their two products, then meet the code — common/math/vec3.hpp — that we’ll use for the rest of the course. The demo chapters/02-vectors/main.cpp shows all of it moving.

Arrows: position, displacement, velocity

The same three numbers (x,y,z) can mean different things depending on how you read them:

  • a position — an arrow from the origin to a point in space;
  • a displacement — “move 3 right and 2 up”, with no fixed start;
  • a velocity — a displacement per second.

Subtraction is the workhorse: b−a is the arrow that points from a to b. You’ll write that line constantly — “the direction from this body to that one” is normalize(b - a). (To normalize a vector means to keep its direction but shrink or grow it to length 1 — a pure “which way”, with the “how far” removed. We’ll define it properly at the end of the chapter.)

A note on the little symbols

Throughout, |a| means the length (or “magnitude”) of the arrow a — a single positive number. The Greek letter θ (“theta”) is just a name for an angle, here the angle between two arrows. cos and sin are the usual trigonometry functions; the only fact you need is that cos of an angle runs from 1 (arrows pointing the same way) through 0 (a right angle) to −1 (pointing opposite).

Adding and scaling

The dot product — “how aligned?”

The dot product multiplies two vectors and gives back a single number:

a·b = axbx +ayby +azbz = |a| |b| cos θ
Intuition The second form is the useful one. Because of the cos θ, the dot product is largest when the arrows point the same way (θ = 0), zero when they’re perpendicular (θ = 90°), and negative when they oppose. So a single multiply-and-add answers “are these facing the same direction?” — the question behind lighting, “is the contact separating or closing?”, and which side of a plane a point is on.

The cross product — “what’s perpendicular?”

The cross product takes two vectors and produces a third one, at right angles to both:

a×b = ( aybz−azby, azbx−axbz, axby−aybx )

Its length is |a||b|sin θ — the area of the parallelogram the two arrows span — and its direction is given by the right-hand rule.

Where we’ll use it Every flat-shading normal in Chapter 4 is a cross product of two triangle edges. Every torque in Chapter 11 is r×F — a force applied off-center. Every contact tangent direction in Part V comes from crossing the normal with something. If the dot product is the engine’s sense of alignment, the cross product is its sense of rotation.

The code

All of the above is a handful of lines. Here are the two products and normalise, straight from common/math/vec3.hpp:

inline constexpr Real dot(const Vec3& a, const Vec3& b) {
    return a.x * b.x + a.y * b.y + a.z * b.z;
}

inline constexpr Vec3 cross(const Vec3& a, const Vec3& b) {
    return {
        a.y * b.z - a.z * b.y,
        a.z * b.x - a.x * b.z,
        a.x * b.y - a.y * b.x,
    };
}

// A unit-length copy. Note the guard: a zero vector has no direction, so we
// return it unchanged instead of dividing by zero and producing NaNs.
inline Vec3 normalize(const Vec3& v) {
    const Real len2 = lengthSq(v);
    if (len2 < kEpsilon * kEpsilon) return v;
    return v * (Real(1) / std::sqrt(len2));
}
Performance habit worth forming now Use lengthSq (no square root) whenever you only need to compare lengths — “is this body within radius r?” is distanceSq(a,b) < r*r. The square root is comparatively slow and, because it’s monotonic, changes nothing about the comparison.

Takeaways

  • A vector is length + direction; b−a points from a to b.
  • Dot → a number, alignment, projection, sides of a plane.
  • Cross → a perpendicular vector, area, rotation axes, normals, torque.
  • Prefer lengthSq to length for comparisons.
