Impulses & Restitution

A contact tells us two bodies overlap. Now we make them do something about it. The trick is the impulse — a sudden kick to velocity that can stop a collision in a single step, where a gentle force never could. And by choosing how much bounce that kick returns, we get everything from a dead thud to a superball.

Goal

Understand impulses and restitution, and how one impulse at a contact changes both a body's motion and its spin. This is our first use of the full World (physics/world.hpp) and the solver (physics/solver.hpp). Demo: chapters/16-impulses/main.cpp.

Why a force isn't enough

Intuition A ball moving down at 5 m/s hits the floor. In one 1/120-second step, a force can only nudge its velocity a little — so next step it's still moving down, now inside the floor. To reverse the velocity this step you'd need an enormous force acting for an instant. That's exactly what an impulse is: a force integrated over time, J=F Δt, treated as a single instantaneous kick. An impulse changes momentum directly, J=Δp=m Δv, so it can stop or reverse a collision in one go.

The normal impulse

At a contact we care about the relative velocity along the normal — how fast the two surfaces are closing. Call it vn. If they're approaching (vn<0), we apply just enough impulse along the normal to cancel that closing speed — and, if the surface is bouncy, to send some of it back:

vn′ =−e vn

where e is the restitution (0 to 1). The impulse magnitude that achieves this depends on the bodies' masses and their resistance to spinning at that point (the "effective mass" — a unit impulse there actually changes the relative velocity by less if it also has to twist the bodies). The solver computes it; the shape of the answer is "how much velocity change we want, divided by how hard it is to move here."

One impulse, two effects

Because a contact is usually off-centre, the impulse does double duty: it changes the body's straight-line velocity and its spin — exactly the r×J torque idea from Chapter 11, but delivered instantly. That's why a box clipped at a corner tumbles. Our RigidBody::applyImpulse is that whole idea in three lines:

void applyImpulse(const Vec3& impulse, const Vec3& worldPos) {
    if (isStatic()) return;
    linearVelocity  += impulse * invMass;                  // straight-line kick
    angularMomentum += cross(worldPos - position, impulse);  // ...and the spin, r × J
}

The solver's core

The normal impulse, accumulated and clamped so a contact never pulls:

const Real vn = dot(A.velocityAtPoint(p) - B.velocityAtPoint(p), normal);
Real dLambda = (velocityBias - vn) * normalMass;      // bias carries the restitution
const Real old = normalImpulse;
normalImpulse = std::max(0.0f, old + dLambda);        // clamp the TOTAL to ≥ 0
dLambda = normalImpulse - old;
A.applyImpulse(normal * dLambda, p);
B.applyImpulse(-normal * dLambda, p);
Why clamp the accumulated total, not each step? A single iteration might want a negative correction (to undo an earlier over-push), which is fine — as long as the running total of the normal impulse stays non-negative. Clamping the accumulated value lets the solver freely adjust while still guaranteeing the contact only ever pushed, never pulled.

Experiments

  • In the widget, set e=1: the ball bounces back to its start height forever (no energy lost). Set e=0: it sticks on first contact. Real materials live in between.
  • Scene 3 bounces boxes. Watch them pick up spin when they land on a corner — one impulse, both effects.

Takeaways

  • An impulse changes momentum instantly, so it can stop a collision in one step where a force can't.
  • The normal impulse cancels the closing speed and returns a fraction e (restitution) of it; it's clamped so a contact only ever pushes.
  • An off-centre impulse changes velocity and spin (r×J).
